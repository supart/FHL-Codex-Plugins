# 更新说明 v0.2.0

本版本是 `fhl-image-gen` 的 v0.2.0 发布整理版，用于从 v0.1.1 升级到新的 FHL Images API 默认链路。

## 相比 v0.1.1 的主要变化

- v0.2.0 的默认生图链路调整为 FHL Images API。
- 文生图、单图图生图、多参考图图生图都默认走 FHL Images API。
- Responses 路线仍保留为显式排查入口，可用 `--api-mode responses` 或 `--fhl-api-mode responses` 指定。
- 固定公开请求策略仍为 2K 比例矩阵，不开放任意 `--size` 自定义。
- 保留最多 10 个 FHL worker 的配置能力，但图生图并发策略更保守。

## 图生图稳定性规则

- 单参考图图生图可以使用高并发；本轮测试中 10 并发成功率达到 `9/10`。
- 多参考图图生图建议 `1..5` 张参考图，并采用顺序或低并发。
- `6..10` 张参考图保留为重任务/诊断范围，不作为稳定公开能力承诺。
- 4 张参考图曾在单 API、单任务模式下复现成功，但高并发不稳定。
- 10 张参考图多次遇到 `524/502/503/fetch failed`，建议改为分组生成中间图后再二次合成。

## 新增或保留能力

- 多 worker 配置与调度保留：最多 10 个 FHL worker。
- 普通文生图、批量文生图、连续出图继续支持自适应调度。
- 通用 `workflow-batch-edit` 保留，适合固定参考图 + 变量图 + 自定义模板的批量生产任务。
- `preset nail-tryon` 保留为兼容预设，但新任务建议优先使用通用 workflow。
- 保存策略保持不变：上游返回原图必须原样落盘，`--resize` 只会生成额外派生副本。

## 使用建议

- 普通文生图：可继续使用高并发。
- 单图图生图：可尝试高并发，但遇到上游 `524/503` 时应降速或稍后重试。
- 多参考图图生图：优先顺序或低并发，不建议 10 并发。
- 多角色或多产品合成：优先分组生成，避免一次请求塞入太多参考图。

## 升级注意

- 升级后不需要重新写入 API Key；API Key 仍应保存在本机 Codex 配置中，不应写入仓库。
- 如果旧配置里只有一个 FHL API Key，插件会继续兼容。
- 发布包不包含任何真实 API Key、本机素材、生成产物或测试图片。
- 若上游 Images API 出现异常，可显式切换到 Responses 路线排查，但默认仍使用 Images API。

## 校验建议

```powershell
node "plugins\fhl-image-gen\scripts\generate.mjs" --get-config
node "plugins\fhl-image-gen\scripts\generate.mjs" --help
node --check "plugins\fhl-image-gen\scripts\generate.mjs"
```

成功标准：

- `--get-config` 只显示脱敏配置，不打印完整 API Key。
- 普通 `--help` 不出现内部诊断或非公开 provider 信息。
- `node --check` 无语法错误。
