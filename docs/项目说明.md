# 项目说明

## 这个仓库是什么

这是一个给 Codex 使用的 Git marketplace 仓库，目前只提供一个插件：

- `fhl-image-gen`

marketplace 信息：

- 名称：`fhl-plugins`
- 展示名：`FHL Plugins`

## 仓库结构

```text
.
├─ .agents/
│  └─ plugins/
│     └─ marketplace.json
├─ plugins/
│  └─ fhl-image-gen/
│     ├─ .codex-plugin/
│     │  └─ plugin.json
│     ├─ assets/
│     ├─ scripts/
│     │  └─ generate.mjs
│     └─ skills/
│        └─ fhl-image-gen/
│           └─ SKILL.md
├─ docs/
│  ├─ 更新说明-v0.1.1.md
│  └─ 项目说明.md
├─ README.md
├─ VERSION.txt
└─ .gitignore
```

## 关键文件作用

### `.agents/plugins/marketplace.json`

定义 marketplace 名称、展示名，以及本仓库中登记的插件列表。

### `plugins/fhl-image-gen/.codex-plugin/plugin.json`

插件元数据，包括：

- 插件名
- 版本号
- marketplace 中显示的描述
- logo / composer icon

### `plugins/fhl-image-gen/scripts/generate.mjs`

插件核心脚本，负责：

- 配置读取与写入
- 文生图
- 图生图
- 多 worker 调度
- 批量任务编排
- 工作流补洞与结果清单输出

### `plugins/fhl-image-gen/skills/fhl-image-gen/SKILL.md`

给 Codex 的技能说明文件，用来告诉模型这个插件应该怎么触发、如何调用、支持哪些参数与限制。

### `README.md`

对普通使用者的中文说明文档。

### `docs/更新说明-v0.1.1.md`

记录 `0.1.0 -> 0.1.1` 的主要变化。

### `docs/项目说明.md`

解释仓库结构和文件职责，也就是你现在看到的这份文档。

## 对外发布内容

公开仓库与发布归档只保留对使用者有价值的内容：

- marketplace 配置
- 插件主体代码
- 公开文档
- 版本信息

不会放进公开仓库的内容：

- 真实 API Key
- 本机私有路径素材
- 运行日志
- 生图产物
- 本地缓存
- 仅用于内部实验的临时文件

## 版本管理方式

当前项目建议按两层管理：

### 1. 工作仓库

用于继续开发、验证、实验和同步 GitHub 更新。

### 2. 版本归档

用于冻结某个稳定版本，方便回溯和复现。归档目录不再继续开发，也不保留 `.git`。

## 当前版本定位

`0.1.1` 的定位是：

- 把 FHL Responses API 稳定链路固化下来
- 把多 worker 调度正式做进插件
- 把通用批量图生图工作流抽象出来
- 把公开仓库整理成可直接给别人安装和使用的状态

## 使用者需要知道的最少信息

如果你只是来安装插件，记住这两条命令就够了：

```bash
codex plugin marketplace add supart/FHL-Codex-Plugins
codex plugin add fhl-image-gen@fhl-plugins
```

更完整的使用方式，请看仓库根目录 [README.md](../README.md)。
