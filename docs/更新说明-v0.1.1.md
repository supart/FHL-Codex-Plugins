# 更新说明 v0.1.1

## 版本概览

`0.1.1` 是在 `0.1.0` 基础上的一次能力升级和对外发布整理版本，重点是把插件推进到可做批量生产流的状态。

## 0.1.0 -> 0.1.1 的主要变化

### 1. 多 worker 池正式加入

- 配置模型从单 `apiKey` 升级为 `workers[]`
- 最多支持 `10` 个独立 worker
- 保留 `--set-key` 兼容入口
- 新增：
  - `--list-workers`
  - `--add-worker-key`
  - `--set-worker-key`
  - `--remove-worker`
  - `--enable-worker`
  - `--disable-worker`

### 2. 图生图链路固定为 Responses API

- 文生图默认走 `POST /v1/responses`
- 图生图默认也走 `POST /v1/responses`
- 使用 `input_text + input_image + image_generation(action:"edit")`
- 不再把旧版 Images API 作为默认编辑链路

这次版本的目标很明确：把真正稳定的链路固化下来。

### 3. 新增通用 `workflow-batch-edit`

这是 `v0.1.1` 的核心升级。

它把之前在真实生产任务里跑通的经验抽象成了通用工作流：

- 固定参考图可重复传入
- 变量图目录可批量展开
- 多模板场景可自由定义
- 每个变量图 × 每个模板 = 一个独立任务
- 任务支持多 worker 并行调度

适用场景不再限定于美甲：

- 模特 + 多服装
- 人物 + 多商品
- 品牌 + 多海报素材
- 角色 + 多配件
- 家具 + 多空间图

### 4. `preset nail-tryon` 内置为预设

美甲试戴链路保留，但从“唯一工作流”降级为“预设和兼容入口”：

- 通用入口：`--workflow-batch-edit`
- 兼容预设：`--preset nail-tryon`
- 旧命令：`--nail-stress-test`

这样既保留了真实生产经验，也避免把插件逻辑写死在单一产品类型上。

### 5. 自动续跑、补洞、会话记录

`workflow-batch-edit` 现在支持：

- 已有有效 PNG 自动跳过
- 扫描缺图后自动补洞
- 输出 `manifest.json`
- 输出 `summary.csv`
- 输出 `failures.json`
- 输出 `sessions.json`

这让批量任务的追踪、复盘和二次补跑都更轻松。

### 6. 比例能力收敛为“已验证集合”

当前版本固定使用 2K 请求矩阵，只开放已验证可用比例：

- `1:1`
- `3:2`
- `2:3`
- `4:3`
- `3:4`
- `16:9`
- `9:16`
- `2:1`
- `1:2`
- `7:4`
- `4:7`

已禁用：

- `5:4`
- `4:5`
- `3:1`
- `1:3`

原因是这些比例在真实 FHL 请求中多次触发上游 `502`，所以直接在插件层禁用，避免误用。

### 7. 对外发布包纯净化

这次还同步做了公开仓库和发布包整理：

- 升级版本号到 `0.1.1`
- README 改成完整中文说明
- 新增 `docs/更新说明-v0.1.1.md`
- 新增 `docs/项目说明.md`
- 清理私有本机路径示例
- 不包含真实 API Key
- 不携带本地缓存、日志、生成产物

## 已知限制

- 当前固定按 2K 比例矩阵请求
- 不开放任意 `--size`
- worker 并行只用于多个独立任务，不会让多个 worker 协作生成同一张图
- FHL 上游不稳定时仍可能出现 `502 / 503 / 504 / 524`，插件会优先做重试、降速和补洞

## 建议升级方式

如果你已经安装过旧版 marketplace：

```bash
codex plugin marketplace upgrade fhl-plugins
codex plugin add fhl-image-gen@fhl-plugins
```

如果你是首次使用，直接看仓库根目录 `README.md` 即可。
